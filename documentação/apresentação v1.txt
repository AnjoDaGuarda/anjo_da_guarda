*Objetivo

O anjo da guarda é um projeto que tem como objetivo avisar ao dono fenômenos que ocorrem em sua residência.

*Hardwares e dispositivos usados:

1 placa arduino
2 sensores (1 movimento e 1 fogo)
1 cabo USB
1 led azul
1 buzzer
1 protoboard
1 computador (usuário)
1 celular (receptor das mensagens e das notificações pelo servidor)



*Bibliotecas:
1 RxtxComm: possibilita a transmição de dados do arduino para a aplicação java e foi a única que achamos que fizesse isso.
2 NexmoSDK: permite a aplicação java enviar mensagens de texto para um celular, foi a única que consiguimos por para funcionar e que fosse FREE.
3 httpcomponents-client-4.3.6: biblioteca da APACHE que permite enviar dados da aplicação java para um servidor. Escolhemos essa biblioteca pela sua robustez e a confiabilidade que a mesma tem no mercado.
4 org.json: para ler conteúdo json do servidor, uma das mais utilizadas no mercado, o que possibilitou maior esclarecimento sob documentações.

*aplicações:
1 aplicação java
1 aplicação android
1 aplicação para arduino
1 página html com suas funcionalidades de servidor em php
1 banco de dados associado ao php (MySQL)

*linguagem de desenvolvimento
Desktop: Java 8, foi escolhida por ser multe plataforma e pela facilidade dos integrantes programarem de forma rápida e minimisando os erros, também foi escolhida pelas suas novas APIs como por exemplo a nova API de data, que pode ser encontrada no pacote java.time e que facilitou muito a captura e o tratamento da data e hora onde os eventos ocorrem.
Arduino:
Web: html e php, para melhor comunicação do banco com o servidor e que era de fácil implementação.
Banco de Dados: MySQL foi usado para fazer a persistência dos dados, para uso do servidor e aplicação android, foi escolhido pela simplicidade e pela sua larga usabilidade na web, só temos uma tabela para guardar as tuplas de eventos, que guardam 4 atributos (id, imei, comando, data);
Sistema de gerenciamento de controle: Git foi escolhido para manter a consistência do projeto e para integrar as atividades dos membros da equipe, visto que a programação foi distribuida, logo os membros não poderiam se encontrar presencialmente e trabalhar juntos em somente uma máquina, nosso repositório está no Github e esta disponível em https://github.com/AnjoDaGuarda/anjo_da_guarda

*arquitetura de projeto do software desktop

Stratege: a aplicação não trata na camada mais alta de abstração envio por sms e envio para o servidor de formas diferentes.
Facade: para diminuir a dependencia do domínio da aplicação do domínio de negócio, gerando um subsistema mais reutilizável.
